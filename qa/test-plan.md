## 一、测试范围

### 1.1 功能测试

| 模块 | 测试要点 |
|------|---------|
| 工具列表展示 | 工具卡片正常渲染，名称/简介/标签/链接均完整显示 |
| 标签筛选 | 点击标签可正确过滤出对应分类的工具，支持多标签筛选（如有） |
| 搜索 | 关键词搜索命中正确，无结果时有友好提示 |
| 跳转链接 | 点击跳转链接可在新标签页打开目标 URL，URL 无死链 |
| 数据加载 | 首次加载、刷新加载、空数据状态均正常 |

### 1.2 界面适配

| 场景 | 要求 |
|------|------|
| 桌面端（≥1280px） | 卡片网格 ≥3 列，布局不拥挤 |
| 平板端（768–1279px） | 卡片 2 列，内容不截断 |
| 移动端（<768px） | 卡片单列，标签可横向滚动或换行，按钮可点击（≥44px 触摸区域） |

### 1.3 兼容性

| 维度 | 覆盖范围 |
|------|---------|
| 浏览器 | Chrome（最新 2 版）、Safari（最新 2 版）、Firefox（最新版）、Edge（最新版） |
| 操作系统 | macOS、Windows 10/11、iOS Safari、Android Chrome |
| 网络 | Wi-Fi / 4G / 弱网（3G 模拟，延迟 ≥1s） |

## 二、测试用例

### TC-01：工具列表正常加载与展示

- **前置条件**：已部署测试环境，工具数据 ≥10 条
- **步骤**：
  1. 打开首页
  2. 观察工具卡片列表
- **预期**：所有工具卡片均正常渲染；每张卡片包含名称、简介、至少 1 个标签、可点击的跳转链接；无空白卡片或信息缺失

### TC-02：标签筛选功能

- **前置条件**：页面加载完成，存在"聊天""绘图"等标签
- **步骤**：
  1. 点击标签「聊天」
  2. 观察列表变化
  3. 再次点击「聊天」取消筛选
- **预期**：选中「聊天」后仅显示该分类工具；取消后恢复全量；筛选结果为 0 时显示空状态提示

### TC-03：搜索功能

- **前置条件**：页面加载完成
- **步骤**：
  1. 在搜索框输入"ChatGPT"
  2. 观察结果
  3. 清空搜索框
- **预期**：搜索结果仅包含名称或简介含"ChatGPT"的工具；清空后恢复全量列表；输入不存在的关键词时显示"未找到相关工具"

### TC-04：跳转链接有效性

- **前置条件**：页面加载完成
- **步骤**：
  1. 逐个点击每个工具的跳转链接
  2. 检查打开的页面
- **预期**：所有链接均在新标签页打开；目标页面可正常访问（HTTP 200）；无 404 或重定向到错误页面

### TC-05：移动端布局适配

- **前置条件**：Chrome DevTools 切换至 iPhone 14 视口（390×844）
- **步骤**：
  1. 打开首页
  2. 检查卡片布局、标签排列、按钮可点击性
- **预期**：卡片单列排列，文字不溢出；标签不遮挡内容；链接/按钮触摸区域 ≥44×44px；页面可正常上下滚动

### TC-06：弱网环境加载

- **前置条件**：Chrome DevTools Network 限速为 Slow 3G
- **步骤**：
  1. 清除缓存后打开首页
  2. 观察加载过程
- **预期**：页面在 10s 内完成渲染（骨架屏或 Loading 状态）；无白屏或 JS 报错；数据加载失败时显示重试提示

### TC-07：浏览器兼容性

- **前置条件**：准备 Chrome、Safari、Firefox、Edge
- **步骤**：
  1. 在每个浏览器打开首页
  2. 执行标签筛选 + 搜索操作
- **预期**：各浏览器下功能表现一致；无布局错乱或 JS 报错；Safari 日期/样式无兼容问题
