## 一、产品概述

### 1.1 产品背景

大模型浪潮席卷全球，AI 工具层出不穷，从对话助手、绘图生成到代码辅助，用户面临"工具选择困难"——到哪里找？哪个适合我？哪个最权威/免费/最新？

AI工具集正是为了解决这个信息不对称问题而生：**一个简洁、直观、可信赖的 AI 工具导航与聚合平台。**

### 1.2 产品定义

**AI工具集**是一个 AI 工具导航网站，核心功能是**聚合展示 + 智能分类 + 一键跳转**，帮助用户快速找到当下最适合自己的 AI 工具，并直接访问使用。

### 1.3 目标用户

| 用户画像 | 描述 |
|---|---|
| **AI 尝鲜者** | 想尝试 AI 工具，但不知道从哪个开始，懒得一个个搜索 |
| **效率工作者** | 希望快速找到某个场景（写文案、做图、代码）下的最佳工具 |
| **技术从业者** | 关注 AI 编程、AI 开发相关工具，需要专业分类导航 |
| **学生/爱好者** | 探索 AI 学习，希望低成本/免费工具推荐 |

### 1.4 核心价值主张

- **省时间**：一站浏览，无需分散搜索
- **有体系**：分类清晰，快速定位
- **可信赖**：人工精选，非简单爬虫罗列
- **轻触达**：点击即跳转，无需注册/登录

## 二、功能清单

### 2.1 核心功能（必做）

| 功能模块 | 功能描述 | 优先级 |
|---|---|---|
| **工具卡片展示** | 每个工具以卡片形式展示：名称、Logo、简介（1-2句）、标签、跳转按钮 | P0 |
| **分类导航** | 按使用场景分类：聊天AI / 绘图生成 / 代码辅助 / 写作助手 / 音视频 / 其他 | P0 |
| **标签筛选** | 支持按标签（免费/付费/开源/国产/支持API等）快速筛选 | P0 |
| **搜索功能** | 支持关键词搜索工具名称/简介 | P0 |
| **工具详情页** | 点击卡片进入详情页，展示完整描述、功能特点、使用限制、官网链接 | P0 |
| **跳转外链** | 一键跳转至工具官网，外部链接标识清晰 | P0 |

### 2.2 增强功能（选做）

| 功能模块 | 功能描述 | 优先级 |
|---|---|---|
| **热门/推荐工具** | 首页展示热门工具和编辑推荐 | P1 |
| **用户收藏** | 支持本地收藏常用工具（LocalStorage） | P1 |
| **工具对比** | 选择多个工具横向对比功能/价格 | P2 |
| **周度更新** | 定期更新工具列表，标注"新增"/"热门"标记 | P1 |
| **评分与评论** | 用户可为工具打分、留简短评论 | P2 |
| **提交工具入口** | 用户可提交未被收录的工具 | P2 |
| **暗色模式** | 支持浅色/深色主题切换 | P1 |
| **移动端适配** | 良好的手机端浏览体验 | P1 |

## 三、用户使用流程

### 3.1 主要场景流程

```
用户打开首页
    ↓
① 快速浏览分类 → 选择感兴趣分类 → 翻阅工具卡片 → 点击跳转
    ↓
② 使用搜索框 → 输入关键词 → 查看搜索结果 → 点击跳转
    ↓
③ 使用标签筛选 → 勾选标签（如"免费"） → 过滤结果展示 → 点击跳转
```

### 3.2 工具详情流程

```
点击工具卡片 → 进入详情页 → 阅读完整介绍 → 点击"访问官网"跳转
                                              ↓
                                    可选：收藏 / 返回列表
```

### 3.3 流程示意（文字版）

```
[首页]
├── Banner（价值主张）
├── 搜索栏
├── 热门推荐区
└── 分类卡片（聊天AI / 绘图 / 代码 / 写作 / 音视频 / 其他）
    ↓
[分类列表页]
├── 左侧：分类筛选 + 标签筛选
├── 右侧：工具卡片网格
└── 每个卡片：Logo + 名称 + 简介 + 标签 + 访问按钮
    ↓
[详情页]
├── 工具名称 + Logo
├── 分类标签
├── 详细介绍
├── 功能特点列表
├── 使用限制 / 价格说明
└── 访问官网（主按钮）
```

## 四、竞品简析

### 4.1 国内外竞品

| 竞品 | 网址 | 简介 | 优势 | 不足 |
|---|---|---|---|---|
| **FutureTools** | futuretools.io | 国外最大的 AI 工具目录之一，工具量大 | 收录全、分类细、搜索强 | 英文界面，国内访问慢 |
| **There's An AI For That** | theresanAIforthat.com | 按场景驱动检索，AI 工具发现引擎 | 场景覆盖广、更新快 | 无中文，信息密度高 |
| **Toolify.ai** | toolify.ai | 每日更新 AI 工具排行榜 | 数据量大、时效性好 | 商业化气息重，广告多 |

### 4.2 我们的差异化定位

| 维度 | 竞品 | AI工具集 |
|---|---|---|
| **目标用户** | 全球用户，偏英文 | 中文用户，本地化 |
| **内容质量** | 数量优先 | 精选优先，宁缺毋滥 |
| **内容调性** | 信息罗列为主 | 有编辑视角的推荐 |
| **访问体验** | 多数有广告/追踪器 | 轻量干净，无广告（初期） |
| **更新频率** | 自动爬取为主 | 人工维护，质量可控 |

**小结**：AI工具集不追求"最全"，而追求"最有用"——面向中文用户，做一个好看、好用、可信赖的 AI 工具导航站。

## 五、里程碑计划

### 5.1 版本规划

| 阶段 | 版本 | 目标 | 核心交付物 | 预计周期 |
|---|---|---|---|---|
| **M1 - MVP** | v0.1 | 验证核心价值：工具导航 | 分类展示 + 搜索 + 跳转，20个精选工具 | 2周 |
| **M2 - 完善** | v0.2 | 丰富内容与体验 | 标签筛选 + 详情页 + 收藏 + 暗色模式，50个工具 | 2周 |
| **M3 - 增长** | v1.0 | 正式发布，数据验证 | 热门推荐 + 周更机制 + 工具提交入口，100+工具 | 2周 |
| **M4 - 迭代** | v1.1+ | 用户反馈驱动迭代 | 评分评论 + 工具对比 + 移动端优化 | 待定 |

### 5.2 M1（MVP）详细拆解

**目标**：在 2 周内上线一个可用的、最小化的 AI 工具导航站。

**交付清单：**
- [ ] 首页 + 分类导航（6个分类）
- [ ] 工具卡片列表（20个精选工具）
- [ ] 关键词搜索
- [ ] 工具详情页
- [ ] 外部跳转功能
- [ ] 基础响应式布局（PC 端优先）

**Done 定义：**
> 用户能在 3 步内找到并跳转到任意收录的 AI 工具，无注册、无广告。

## 六、非功能需求（备注）

| 维度 | 要求 |
|---|---|
| **性能** | 首屏加载 < 2s，工具列表页 < 500ms |
| **稳定性** | 外链跳转成功率 > 99% |
| **可维护性** | 工具数据采用 JSON/YAML 管理，便于非技术人员更新 |
| **隐私合规** | 不追踪用户行为，不要求登录（初期） |
